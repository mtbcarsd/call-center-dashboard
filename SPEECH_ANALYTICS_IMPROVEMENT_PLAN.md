# План улучшений речевой аналитики (на основе анализа встречи с конкурентом)

Источник: видеозапись `Встреча речевая аналитика 06.07.2026.mp4` — демо-встреча банка (МТ-банк) с вендором речевой аналитики. Транскрипция выполнена локально (faster-whisper, 74 минуты аудио).

## Что показал конкурент

- **Чек-листы**: кастомные весовые категории, LLM/copilot сам пишет промпт под категорию, поддержка морфологии слов, инверсия категорий («выполнили всё, кроме приветствия»)
- **Паузы/тишина в диалоге**: настраиваемый порог (например, 5 сек), льготный период для категорий вроде «оставайтесь на линии» — конкурент заявляет снижение показателя тишины на 25% после внедрения
- **Диаризация** (разделение оператор/клиент), в т.ч. для моно-записей (как у банка через Cisco)
- **Детекция негатива по позиции в звонке** (начало/середина/конец) — отдельная ML-модель, обученная на сотнях часов размеченных данных
- **AI-копайлот**: запрос на естественном языке → фильтр («покажи звонки Иванова за вчера»)
- **Гибкий конструктор графиков/дашбордов** с drill-down в конкретные звонки
- **Ручной override оценки** с пометкой «M» (manual) и историей изменений, тред комментариев супервайзер ↔ оператор
- **Геймификация**: личный рейтинг оператора среди коллег
- **Автоотчёты и рекомендации по оператору** через LLM
- **Тренировочный бот** (цифровой клиент + суфлёр) для обучения операторов
- Голосовые боты, обзвон, аудио-бейджи для очного обслуживания — не в тему колл-центра-дашборда, из плана исключены

## План улучшений (с учётом стека: Streamlit + Snowflake + Whisper + Ollama)

### Уровень 1 — быстро, высокая ценность

1. **Пауза/тишина в диалоге** — уже есть таймкоды сегментов Whisper, нужно посчитать gaps между репликами → % тишины от длительности звонка.
2. **Чек-лист вместо жёстких полей** — заменить фиксированный `ANALYSIS_PROMPT` в `pipeline.py` на конфигурируемый набор категорий (YAML/JSON: название, вес, промпт) → взвешенный score вместо одной оценки от LLM.
3. **Диаризация оператор/клиент** — критично, почти всё остальное (кто перебивал, доля речи, где негатив) зависит от разделения каналов. Если записи стерео с раздельными каналами — тривиально; если моно — нужна pyannote.audio.
4. **Позиция негатива в звонке** — делить транскрипт на трети/четверти, отдельным полем в LLM-промпте помечать, в какой части диалога был негатив.
5. **Поиск и фильтр по тексту/темам** в `dashboard.py` — сейчас только selectbox по department/urgency/status, добавить полнотекстовый поиск и фильтр по key_topics.

### Уровень 2 — среднесрочно

6. **Ручной override + комментарии супервайзера** — новая таблица в Snowflake (manual_scores/review_comments), UI-виджет с историей правок.
7. **Ролевая модель** — сейчас один пользователь Julia/admin; добавить роль supervisor (видит отдел) и operator (видит только свои звонки).
8. **Конструктор графиков** — вместо хардкод-графиков дать selectbox «метрика × группировка × агрегация» на Plotly.
9. **LLM-копайлот по запросам** — промпт к ollama, который переводит фразу типа «покажи звонки Иванова за вчера» в фильтр по df.
10. **Рейтинг/геймификация операторов** — простой leaderboard-таб со средним score и динамикой по месяцам.

### Уровень 3 — вне текущего масштаба пилота

11. Голосовые боты, суфлёр в реальном времени, обзвон — требуют интеграции с телефонией в реальном времени, не входит в задачи дашборда.
12. Масштабирование (очередь задач, GPU, Kubernetes) — актуально при переходе с 10 тестовых файлов на прод-объём.

## Рекомендуемый старт

Начать с пунктов 1–3 уровня 1 (тишина, чек-лист, диаризация) — дёшевы в реализации и дают наибольший эффект в сравнении с тем, что видели на демо конкурента.
